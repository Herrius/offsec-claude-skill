# Reporting Checklist

Essential components for professional penetration testing reports.

---

## Report Structure

### Front Matter
- [ ] Cover page with client name and date
- [ ] Table of contents
- [ ] Document version control
- [ ] Classification markings (if required)
- [ ] Distribution list

### Executive Summary
- [ ] High-level overview (non-technical)
- [ ] Key findings summary
- [ ] Risk assessment overview
- [ ] Critical vulnerabilities highlighted
- [ ] Business impact described
- [ ] Strategic recommendations
- [ ] Overall security posture assessment

### Methodology
- [ ] Testing approach described
- [ ] Standards followed (OWASP, PTES, etc.)
- [ ] Tools used listed
- [ ] Testing timeline provided
- [ ] Scope reiterated
- [ ] Limitations noted

---

## Technical Findings

### Vulnerability Template
For each finding, include:
- [ ] Unique identifier/reference number
- [ ] Vulnerability title
- [ ] Severity rating (Critical/High/Medium/Low/Info)
- [ ] CVSS score (if applicable)
- [ ] CWE/CVE references (if applicable)
- [ ] Affected systems/components
- [ ] Detailed description
- [ ] Technical impact
- [ ] Business impact
- [ ] Proof of concept
- [ ] Reproduction steps
- [ ] Evidence (screenshots, logs, code)
- [ ] Remediation recommendations
- [ ] References

### Severity Definitions
- [ ] Critical: Immediate exploitation possible
- [ ] High: Significant security risk
- [ ] Medium: Notable security concern
- [ ] Low: Minor security issue
- [ ] Informational: No immediate risk

---

## Evidence Collection

### Screenshots
- [ ] Include timestamps
- [ ] Show full context
- [ ] Highlight important elements
- [ ] Include URL/system identifiers
- [ ] Redact sensitive information if needed

### Code Samples
- [ ] Syntax highlighting applied
- [ ] Comments explaining key parts
- [ ] Before/after comparisons
- [ ] Payload examples

### Network Captures
- [ ] Packet captures included
- [ ] Relevant packets highlighted
- [ ] Decoded/analyzed output

### Logs
- [ ] Relevant log entries extracted
- [ ] Timestamps included
- [ ] Context provided

---

## Remediation Guidance

For each vulnerability:
- [ ] Specific fix provided
- [ ] Code examples (if applicable)
- [ ] Configuration changes detailed
- [ ] Architectural recommendations
- [ ] Priority/timeline suggested
- [ ] Validation steps included
- [ ] References to security best practices

---

## Risk Assessment

### Summary Statistics
- [ ] Total vulnerabilities by severity
- [ ] Vulnerabilities by category
- [ ] Risk score calculation
- [ ] Comparison to previous assessments (if applicable)

### Risk Matrix
- [ ] Visual representation of risks
- [ ] Impact vs likelihood plotted
- [ ] Color-coded severity levels

## Appendices

- [ ] Detailed technical data
- [ ] Full vulnerability list
- [ ] Tool output samples
- [ ] Network diagrams
- [ ] Port scan results

## Quality Assurance

### Technical Review
- [ ] All findings verified
- [ ] False positives removed
- [ ] Severity ratings justified
- [ ] Remediation advice validated
- [ ] Technical accuracy confirmed

### Editorial Review
- [ ] Grammar and spelling checked
- [ ] Formatting consistent
- [ ] Screenshots legible
- [ ] References accurate
- [ ] Acronyms defined
- [ ] Professional tone maintained

### Compliance
- [ ] Client requirements met
- [ ] Contractual deliverables included
- [ ] Required sections present
- [ ] Proper classification applied
- [ ] Distribution restrictions noted

## Report Template Sections

```

   - Tools

   - Scope

3. Technical Findings

   - High Vulnerabilities
   - Medium Vulnerabilities
   - Low Vulnerabilities
   - Informational Items

4. Risk Assessment
   - Risk Matrix
   - Impact Analysis
   - Likelihood Assessment

5. Remediation Roadmap
   - Short-term (0-30 days)
   - Medium-term (30-90 days)
   - Long-term (90+ days)

6. Conclusion
   - Summary
   - Next Steps
   - Contact Information

7. Appendices
   - Technical Details
   - Tool Output
   - Evidence Archive
```
