# Pre-Engagement Checklist

Essential steps before starting a penetration test.

---

## Authorization & Legal

- [ ] Signed Statement of Work (SOW) or contract
- [ ] Signed Rules of Engagement (RoE)
- [ ] Written authorization from authorized personnel
- [ ] Non-Disclosure Agreement (NDA) if required
- [ ] Insurance verification
- [ ] Emergency contact information obtained
- [ ] Legal review completed

---

## Scope Definition

- [ ] In-scope targets clearly defined (IPs, domains, systems)
- [ ] Out-of-scope targets explicitly listed
- [ ] Geographical restrictions identified
- [ ] Time windows for testing established
- [ ] Types of testing authorized (web, network, social engineering, physical)
- [ ] Testing depth/intensity agreed upon
- [ ] Red team vs penetration test clarified

---

## Technical Preparation

### Environment Setup
- [ ] Kali Linux or testing VM ready
- [ ] VPN or testing network access configured
- [ ] All required tools installed and updated
- [ ] Backup systems in place
- [ ] Secure storage for findings ready

### Tools Verification
- [ ] Nmap
- [ ] Burp Suite
- [ ] Metasploit Framework
- [ ] Custom scripts tested
- [ ] Python environment configured
- [ ] Wordlists downloaded and ready

### Infrastructure
- [ ] Attack infrastructure deployed (if needed)
- [ ] C2 server configured (if authorized)
- [ ] Domain names registered (if needed)
- [ ] Phishing infrastructure (if authorized)

---

## Communication

- [ ] Primary point of contact identified
- [ ] Communication channels established (email, phone, chat)
- [ ] Escalation procedures defined
- [ ] Incident reporting process agreed
- [ ] Daily/weekly status update schedule set
- [ ] Final presentation date scheduled

---

## Risk Management

- [ ] Backup and recovery procedures reviewed
- [ ] Production system testing protocols defined
- [ ] DoS testing explicitly authorized or forbidden
- [ ] Data handling procedures established
- [ ] Privacy requirements understood
- [ ] Compliance requirements noted (PCI-DSS, HIPAA, etc.)

---

## Documentation

- [ ] Note-taking system ready
- [ ] Screenshot tool configured
- [ ] Video recording capability (if needed)
- [ ] Evidence storage location prepared
- [ ] Report template prepared
- [ ] Chain of custody procedures established

---

## Logistics

- [ ] Testing dates confirmed
- [ ] Team assignments (if team engagement)
- [ ] On-site access arranged (if physical)
- [ ] Travel arrangements (if required)
- [ ] Equipment checklist completed
- [ ] Budget approved

---

## Kickoff Meeting

- [ ] Kickoff meeting scheduled
- [ ] All stakeholders invited
- [ ] Agenda prepared
- [ ] Questions list prepared
- [ ] Goals and objectives clarified
- [ ] Success criteria defined

---

## Final Checks

- [ ] All documentation reviewed
- [ ] All parties have signed necessary agreements
- [ ] Testing infrastructure verified
- [ ] Team ready to proceed
- [ ] Client ready to proceed
- [ ] Go/No-go decision made

---

## Emergency Procedures

- [ ] Emergency shutdown procedure defined
- [ ] Incident response plan reviewed
- [ ] Insurance contact information available
- [ ] Legal counsel contact available
- [ ] Client IR team contact information
